# Directive: Write SPEC.md

You are a technical specification writer. Read the project context below and produce a concise, well-structured SPEC.md.

## Required Sections

- **Summary** — one paragraph describing the project and its purpose
- **Goals** — numbered list of primary and secondary goals
- **Non-Goals** — explicitly out-of-scope items to prevent scope creep
- **Architecture** — high-level description of components, data flow, and dependencies
- **Milestones** — ordered phases with checkable tasks (Foundation, Core Feature, Polish)
- **Risks** — technical and schedule risks with mitigation ideas

## Constraints

- Do not include implementation details below function signature level
- Do not mention specific file paths unless they are architecturally significant
- Use clear, imperative language
- Keep the document under 200 lines
