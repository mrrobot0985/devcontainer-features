# Creating Your First Feature

This tutorial shows how to add a new feature to this monorepo, write its metadata and install script, and test it locally.

## Step 1: Create the feature directory

Every feature lives under `src/<feature-id>/`. Create a new directory:

```bash
mkdir -p src/hello-feature
cd src/hello-feature
```

A feature must contain at least:

- `devcontainer-feature.json` — metadata and options.
- `install.sh` — installation script, executable.

Optional files:

- `uninstall.sh` — cleanup logic.
- `README.md` — auto-generated from metadata (do not edit by hand).
- `hooks/`, `lib/`, etc. — supporting files installed or used by `install.sh`.

## Step 2: Write `devcontainer-feature.json`

Create `src/hello-feature/devcontainer-feature.json`:

```json
{
  "id": "hello-feature",
  "name": "Hello Feature",
  "version": "0.1.0",
  "description": "Writes a greeting file into the user's home directory",
  "options": {
    "greeting": {
      "type": "string",
      "default": "Hello from devcontainers",
      "description": "Greeting text to write"
    },
    "loud": {
      "type": "boolean",
      "default": false,
      "description": "Write the greeting in uppercase"
    }
  },
  "installsAfter": [
    "ghcr.io/devcontainers/features/common-utils"
  ]
}
```

Field summary:

- `id` — the feature identifier and the directory name.
- `name` — human-readable name.
- `version` — SemVer version used for release tags.
- `description` — short summary shown in the catalog.
- `options` — user-configurable values.
- `installsAfter` — ordering hint for the devcontainer CLI.

## Step 3: Write `install.sh`

Create `src/hello-feature/install.sh`:

```bash
#!/bin/sh
set -e

echo "Activating feature 'hello-feature'"

GREETING="${GREETING:-Hello from devcontainers}"
LOUD="${LOUD:-false}"
USER_HOME="${_REMOTE_USER_HOME:-$HOME}"

if [ "$LOUD" = "true" ]; then
    GREETING=$(printf '%s' "$GREETING" | tr '[:lower:]' '[:upper:]')
fi

printf '%s\n' "$GREETING" > "$USER_HOME/.hello-feature-greeting"
chown "${_REMOTE_USER:-root}:${_REMOTE_USER:-root}" "$USER_HOME/.hello-feature-greeting"

echo "Wrote greeting to $USER_HOME/.hello-feature-greeting"
```

Important conventions:

- The script runs as `root` during the image build.
- Option values are passed as environment variables named after the option id in uppercase: `greeting` becomes `GREETING`, `loud` becomes `LOUD`.
- Use `_REMOTE_USER_HOME` for the target user's home directory and `_REMOTE_USER` for their username.
- `set -e` makes the feature fail the build if any command errors.

Make the script executable:

```bash
chmod +x src/hello-feature/install.sh
```

## Step 4: Generate the feature README

Run the README generator from the repository root:

```bash
uv run python scripts/generate-feature-readmes.py
```

This creates `src/hello-feature/README.md` from the JSON metadata. If you update options later, rerun the generator.

## Step 5: Test the feature locally

Build and test the feature with the Dev Container CLI against a common base image:

```bash
devcontainer features test \
    --skip-scenarios \
    -f hello-feature \
    -i mcr.microsoft.com/devcontainers/base:ubuntu \
    .
```

If this is your first test, there are no scenario tests yet, so `--skip-scenarios` is appropriate. The CLI will create a container with the default option values and run a default smoke test.

After the build finishes, open a shell in the generated container and verify the result:

```bash
cat ~/.hello-feature-greeting
# -> Hello from devcontainers
```

## Step 6: Add scenario tests (optional)

For non-trivial features, add tests under `test/hello-feature/`:

```bash
mkdir -p test/hello-feature
```

- `test/hello-feature/scenarios.json` — defines named scenarios with option overrides.
- `test/hello-feature/test.sh` — default test script.
- `test/hello-feature/<scenario>.sh` — per-scenario test script.

Example `scenarios.json`:

```json
{
    "loud_greeting": {
        "image": "mcr.microsoft.com/devcontainers/base:ubuntu",
        "features": {
            "hello-feature": {
                "greeting": "hi there",
                "loud": true
            }
        }
    }
}
```

Example `test/hello-feature/loud_greeting.sh`:

```bash
#!/bin/bash
set -e

greeting=$(cat ~/.hello-feature-greeting)
if [ "$greeting" != "HI THERE" ]; then
    echo "Expected uppercase greeting, got: $greeting"
    exit 1
fi
```

Run scenario tests with:

```bash
devcontainer features test -f hello-feature --skip-autogenerated .
```

## Step 7: Run the local CI gate

Before opening a pull request, run the local gate:

```bash
./scripts/local-ci.sh
```

It runs shellcheck, README sync checks, workflow validation, a dry-run release, and feature smoke tests.
