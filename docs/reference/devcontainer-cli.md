# Dev Container CLI Reference

The Dev Container CLI is the tool that builds dev containers, runs tests, and publishes features. You can install it globally via npm or use the binary bundled with the VS Code Dev Containers extension.

## Install the CLI

```bash
npm install -g @devcontainers/cli
```

Bundled VS Code binary (path may vary by platform):

```bash
~/.config/Code/User/globalStorage/ms-vscode-remote.remote-containers/cli-bin/devcontainer
```

## Build a container from `devcontainer.json`

From the project root:

```bash
devcontainer up --workspace-folder .
```

Force a fresh build and ignore cached layers:

```bash
devcontainer up --workspace-folder . --build-no-cache
```

## Test a single feature locally

Default-install smoke test, skipping scenarios:

```bash
devcontainer features test \
    --skip-scenarios \
    -f claude-code-hooks \
    -i mcr.microsoft.com/devcontainers/base:ubuntu \
    .
```

Run only the scenario tests for one feature:

```bash
devcontainer features test \
    -f claude-code-hooks \
    --skip-autogenerated \
    .
```

Run the global integration scenarios:

```bash
devcontainer features test --global-scenarios-only .
```

### Common flags

| Flag                      | Description                                                         |
| ------------------------- | ------------------------------------------------------------------- |
| `-f <feature>`            | Test a specific feature by id.                                      |
| `-i <image>`              | Base image for the test container.                                  |
| `--skip-scenarios`        | Skip tests defined in `scenarios.json`.                             |
| `--skip-autogenerated`    | Skip the autogenerated default-option test.                         |
| `--skip-duplicated`       | Skip duplicated autogenerated tests when running multiple features. |
| `--project-folder <path>` | Path to the repository root containing `src/` and `test/`.          |

## Remove stale feature caches

If features are not updating after a new release, clean all caches:

```bash
# Remove old containers and images created by the CLI
docker ps -aq --filter label=devcontainer.local_folder | xargs -r docker rm -f
docker images --format "{{.Repository}}:{{.Tag}}" | grep "vsc-" | xargs -r docker rmi -f

# Remove a project-specific volume if the feature created one
docker volume ls -q | grep "claude-code-config" | xargs -r docker volume rm -f

# Remove the devcontainer CLI feature cache
rm -rf /tmp/devcontainercli-*/container-features/*

# Remove a lockfile that may pin an older feature digest
rm -f .devcontainer/devcontainer-lock.json
```

## Important: lockfiles pin feature versions

If `.devcontainer/devcontainer-lock.json` exists, it overrides `:latest`/`:0` and pins each feature to a specific digest. Delete the lockfile to force resolution of the newest published version.

```bash
rm -f .devcontainer/devcontainer-lock.json
```

## Additional useful commands

Build the dev container image without starting it:

```bash
devcontainer build --workspace-folder .
```

Rebuild everything from scratch and raise the log level:

```bash
devcontainer up --workspace-folder . --build-no-cache --log-level debug
```
